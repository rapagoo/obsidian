# C++ 문자열/입출력/자료구조 기초 정리

이 문서는 **HackerRank Attribute Parser** 같은 문제를 풀기 전에 알아야 하는 C++ 기본 도구들을 정리한 복습용 노트이다.

핵심은 함수 암기가 아니라,

- 한 줄 전체를 읽기
- 문자열 일부를 잘라내기
- 문자열을 토큰으로 분리하기
- key-value 형태로 저장하기

를 익히는 것이다.

---

## 1. `getline`

### 역할
한 줄 전체를 통째로 읽는다.

### 기본 형태
```cpp
string line;
getline(cin, line);
```

### 왜 필요한가
`cin >>` 는 공백 전까지만 읽는다.

예를 들어:
```cpp
string s;
cin >> s;
```

입력이
```text
<tag1 value = "HelloWorld">
```
라면 `s` 에는
```text
<tag1
```
만 들어간다.

하지만
```cpp
string line;
getline(cin, line);
```
를 쓰면
```text
<tag1 value = "HelloWorld">
```
전체가 들어간다.

### 기억할 점
- `cin >>` : 공백 전까지만 읽음
- `getline` : 한 줄 전체를 읽음

### 예제
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string line;
    getline(cin, line);
    cout << line << '\n';
}
```

### 이 문제와 연결
HRML 한 줄 전체를 읽어야 하므로 `getline`이 필요하다.

## 2. `ignore`

### 역할
입력 버퍼에 남아 있는 엔터(`\n`)를 버린다.

### 기본 형태
```cpp
cin.ignore();
```

### 왜 필요한가
예를 들어:
```cpp
int n;
cin >> n;
string line;
getline(cin, line);
```

이 경우 `cin >> n` 뒤에 남아 있는 엔터를 `getline`이 읽어버려서 `line`이 빈 문자열이 될 수 있다.

그래서 보통 이렇게 쓴다.

```cpp
int n;
cin >> n;
cin.ignore();
string line;
getline(cin, line);
```

### 기억할 점
`cin >> ...` 다음에 `getline`을 쓸 때 자주 필요하다.

### 예제
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    int n;
    cin >> n;
    cin.ignore();

    string line;
    getline(cin, line);
    cout << line << '\n';
}
```

### 이 문제와 연결
처음에 `n`, `q` 를 `cin >> n >> q;` 로 읽고, 그 다음 HRML 줄들을 `getline`으로 읽기 때문에 `ignore()`가 필요하다.

## 3. `substr`

### 역할
문자열의 일부를 잘라서 가져온다.

### 기본 형태
```cpp
s.substr(시작위치, 길이)
```

### 예제 1
```cpp
string s = "abcdef";
cout << s.substr(1, 3);
```
출력:
```text
bcd
```

### 예제 2: 양끝 문자 제거
```cpp
string s = "<tag1>";
cout << s.substr(1, s.size() - 2);
```
출력:
```text
tag1
```

### 예제 3: 따옴표 제거
```cpp
string s = "\"HelloWorld\"";
cout << s.substr(1, s.size() - 2);
```
출력:
```text
HelloWorld
```

### 헷갈리기 쉬운 점
코드에서 `\"` 는 **문자열 안에 큰따옴표 하나를 넣기 위한 표기**다.

```cpp
string s = "\"HelloWorld\"";
```
의 실제 문자열 내용은
```text
"HelloWorld"
```
가 아니라
```text
"HelloWorld"
```
처럼 보일 수 있지만, 실제 저장되는 문자는
```text
"HelloWorld"
```
가 아니라
```text
"HelloWorld"
```
도 아니다. 실제로는 **양끝 큰따옴표가 포함된 문자열**이다.

정확히 말하면 실제 내용은:
```text
"HelloWorld"
```
처럼 코드로 쓰는 게 아니라,
```text
"HelloWorld"
```
가 아닌
```text
"HelloWorld"
```
도 아니다. 이 부분이 헷갈리면 직접 출력해보는 것이 가장 좋다.

간단히 기억하면:
- 코드에서 `\"` 라고 써도
- 실제 문자열에는 `"` 하나만 들어간다.

### 이 문제와 연결
- `<` 와 `>` 제거
- 속성값 양끝 따옴표 제거
에 사용된다.

## 4. `find`

### 역할
문자열에서 특정 문자 또는 문자열이 **처음 등장하는 위치**를 찾는다.

### 기본 형태
```cpp
s.find(찾을문자또는문자열)
```

### 예제
```cpp
string s = "tag1.tag2";
cout << s.find('.');
```
출력:
```text
4
```

### 못 찾았을 때
```cpp
if (s.find('.') == string::npos) {
    cout << "점이 없다";
}
```

### 기억할 점
- 찾으면 위치 반환
- 못 찾으면 `string::npos`

### 이 문제와 연결
필수는 아니지만, 구분자 위치를 찾고 문자열을 나눌 때 유용하다.

---

## 5. `rfind`

### 역할
문자열을 **뒤에서부터 찾아서**, 마지막 등장 위치를 찾는다.

### 기본 형태
```cpp
s.rfind(찾을문자또는문자열)
```

### 예제
```cpp
string s = "tag1.tag2.tag3";
cout << s.rfind('.');
```

### 왜 필요한가
현재 경로가
```text
tag1.tag2
```
일 때 닫는 태그를 만나면
```text
tag1
```
로 돌아가야 한다.

즉, 마지막 `.` 의 위치를 찾아 그 앞부분만 남기면 된다.

### 예제
```cpp
string path = "tag1.tag2";
size_t pos = path.rfind('.');
path = path.substr(0, pos);
cout << path;
```
출력:
```text
tag1
```

### 못 찾았을 때
```cpp
if (path.rfind('.') == string::npos) {
    path = "";
}
```

### 기억할 점
- `find` : 앞에서부터
- `rfind` : 뒤에서부터

### 이 문제와 연결
닫는 태그를 처리할 때 현재 경로를 한 단계 위로 올리는 데 사용된다.

## 6. `stringstream`

### 역할
문자열을 `cin` 처럼 공백 기준으로 잘라 읽게 해준다.

### 기본 형태
```cpp
stringstream ss(line);
```

### 예제
```cpp
#include <iostream>
#include <sstream>
#include <string>
using namespace std;

int main() {
    string line = "tag1 value = \"HelloWorld\"";
    stringstream ss(line);

    string a, b, c, d;
    ss >> a >> b >> c >> d;

    cout << a << '\n';
    cout << b << '\n';
    cout << c << '\n';
    cout << d << '\n';
}
```

출력:
```text
tag1
value
=
"HelloWorld"
```

### 왜 필요한가
`getline` 으로 한 줄 전체를 받은 뒤,
그 줄을 다시 공백 기준으로 쪼개서

- 태그명
- 속성명
- `=`
- 속성값

을 읽기 쉽도록 해준다.

### 기억할 점
- `getline` : 한 줄 읽기
- `stringstream` : 한 줄을 다시 토큰으로 분해하기

### 이 문제와 연결
`<`, `>` 를 제거한 뒤 태그명과 속성들을 파싱할 때 사용한다.

---

## 7. `map`

### 역할
key-value 형태로 데이터를 저장한다.

### 기본 형태
```cpp
map<string, string> mp;
```

### 예제
```cpp
map<string, string> mp;
mp["apple"] = "red";
mp["banana"] = "yellow";

cout << mp["apple"] << '\n';
```
출력:
```text
red
```

### 존재 여부 확인
```cpp
if (mp.find("apple") != mp.end()) {
    cout << "있다";
}
```

### 왜 필요한가
이 문제의 질의는 이런 식이다.

```text
tag1.tag2~name
```

따라서 파싱할 때 미리 이렇게 저장하면 된다.

```cpp
mp["tag1~value"] = "HelloWorld";
mp["tag1.tag2~name"] = "Name1";
```

그러면 질의가 들어왔을 때

```cpp
if (mp.find(query) != mp.end()) cout << mp[query] << '\n';
else cout << "Not Found!\n";
```

처럼 바로 답할 수 있다.

### 기억할 점
이 문제에서 `map`은 **정답 사전**이다.

# 이 7개가 문제에서 어떻게 연결되는가

문제 풀이 흐름은 이렇게 정리할 수 있다.

1. `cin >> n >> q;`
2. `cin.ignore();`
3. `getline` 으로 HRML 한 줄 읽기
4. 닫는 태그인지 확인
5. 여는 태그라면 `substr` 로 `<`, `>` 제거
6. `stringstream` 으로 태그명/속성명/속성값 분리
7. 현재 경로와 속성명을 합쳐 `map`에 저장
8. 닫는 태그라면 `rfind` 와 `substr` 로 현재 경로 줄이기
9. 질의는 `map.find()` 로 찾기

즉, 한 줄로 요약하면:

> 한 줄을 읽고 → 필요한 조각을 꺼내고 → key를 만들어 저장하고 → 나중에 그대로 찾는다.

---

# 최소 암기 요약

```cpp
getline(cin, line);              // 한 줄 전체 읽기
cin.ignore();                    // 남은 엔터 제거
s.substr(a, b);                  // 문자열 일부 자르기
s.find(x);                       // 앞에서 찾기
s.rfind(x);                      // 뒤에서 찾기
stringstream ss(line);           // 문자열을 토큰으로 분해
map<string, string> mp;          // key-value 저장
```

# 한 줄 결론

이 문제는 `rfind`, `ignore`, `substr`를 외운 사람만 푸는 문제가 아니라,

**문자열을 조금씩 잘라서 필요한 정보만 꺼내는 사고**를 익힌 사람이 풀 수 있는 문제다.

함수는 그 사고를 구현하는 도구일 뿐이다.
