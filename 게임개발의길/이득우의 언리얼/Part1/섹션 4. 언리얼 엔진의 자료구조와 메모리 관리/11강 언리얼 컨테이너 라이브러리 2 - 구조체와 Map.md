## 구조체 UStruct
데이터 저장/전송에 특화된 가벼운 객체
대부분 GENERATED_BODY 매크로를 선언해준다.
언리얼 엔진의 구조체 이름은 F로 시작함
NewObject API 사용불가
대부분 스택 내 데이터로 사용됨
## TMap의 구조와 활용
https://dev.epicgames.com/documentation/ko-kr/unreal-engine/map-containers-in-unreal-engine?application_version=5.7
TSet을 TPair 로 구성한 자료구조가 TMap 이라서 성질이 동일하다

## 정리
- `TArray`
    - 순서 유지
    - 중복 허용
    - “목록”에 적합
- `TSet`
    - 중복 불가
    - 순서 보장 약함
    - “존재 여부 / 고유값 집합”에 적합
- `TMap`
    - Key → Value
    - Key는 유일
    - “검색용 사전”에 적합
- `TMultiMap`
    - Key → 여러 Value
    - 중복 키 허용
    - “이름 같은 사람 여러 명” 같은 경우에 적합
- `USTRUCT`
    - 관련 데이터 묶기
    - 컨테이너에 넣기 좋음
    - Set/Map에 넣을 때는 `==`, `GetTypeHash` 설계가 중요
