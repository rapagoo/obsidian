# 컴퓨터가 이해하는 정보

컴퓨터는 우리가 사용하는 프로그래밍 언어를 이해하지 못한다.
대신 컴퓨터는 데이터와 명령어를 이해한다.
컴퓨터의 명령어는 수행할 동작과 대상으로 이루어져 있다.
데이터는 정적인 정보를 의미한다(이미지, 숫자 등).
명령어는 이 데이터를 활용하는 정보이다.
즉, 데이터는 명령어에 종속적인 정보이며 명령어의 재료이다.
CPU가 이 명령어들을 실행한다.
# 컴퓨터의 핵심 부품

컴퓨터의 핵심 부품에는
CPU(중앙처리장치), 메모리(주기억장치), 캐시 메모리, 보조기억장치, 입출력장치
가 존재한다.
## CPU
정보를 읽어 들이고, 해석하고, 실행하는 부품이 CPU이다.
CPU의 구성 요소: 산술논리연산장치(ALU), 제어장치(CU), 레지스터
이 중 가장 중요한 요소는 레지스터이다.
레지스터가 바로 명령어를 저장하는 요소이기 때문

## 메모리와 캐시 메모리
CPU가 읽어 들이고 해석하고 실행한 정보가 어딘가에 저장되어 있어야 하는데, 이것이 바로 메모리이다.
메모리는 즉 현재 실행 중인 프로그램을 구성하는 데이터와 명령어를 저장하는 부품이다.
중요한 것은 **실행 중**인 프로그램을 저장한다는 것이다.
메모리에는 주소와 휘발성이라는 개념이 존재한다.
주소는 메모리에 있는 정보에 CPU가 접근하기 위해 필요한 것이다.
또한 RAM은 휘발성 장치이다.
**캐시 메모리** 는 CPU와 메모리 사이에 존재하는, CPU가 메모리에 저장된 값에 빠르게 접근하기 위한 저장장치이다. 
## 보조기억장치
휘발설 저장장치인 메모리를 보조하기 위해 **비휘발성** 저장장치인 보조기억장치가 존재한다.
메모리는 현재 실행 중인 프로그램을 저장하고, 보조기억장치는 보관할 프로그램을 저장한다.
보조기억장치에 있는 프로그램을 실행하려면, 메모리로 복사한 다음 CPU가 실행한 수 있다.
## 입출력장치
입출력장치는 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환하는 장치를 말한다.
## 메인 보드와 버스
컴퓨터의 핵심 부품들이 모두 모여있는 기판을 메인 보드라 부른다.
이 메인 보드에서는 각 컴퓨터 부품들이 정보를 주고받는 통로인 버스가 존재한다.
가장 중요한 버스는 핵심 부품들을 연결하는 시스템 버스이다.
## 저장장치의 계층 구조
CPU와 가까운 저장장치는 빠르고, 멀리 있는 저장장치는 느리다.
속도가 빠른 저장장치는 용량이 작고, 가격이 비싸다.
